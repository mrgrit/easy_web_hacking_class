# 커리큘럼 설계서 — 본 과정 5주차 + 특별 세션 3주

> 각 섹터 = 교과서(`lecture_*.md`) + 실습(`lab_*.yaml`) 한 쌍.
> **모든 실습은 브라우저(F12)만으로 완주 가능**하게 설계한다. `curl` 은 "같은 일을 기계가 하는
> 방식"으로 병기만 하고, 유일한 경로로 쓰지 않는다.

## 한눈에 보기 — 특별 세션 (AI 서비스 모의해킹)

| 세션 | 제목 | 시간 | 표적 |
|------|------|------|------|

| **특별 2** | 프롬프트 인젝션 3종 + 챗봇의 답이 흉기가 될 때 | 5h | AICompanion(mock) |

> **설계 의도** — 가장 노잼이 되기 쉬운 *개론·윤리* 를 맨 앞에 짧고 강렬하게 배치하되,
> 여기에 **요즘 에이전트 지도(claw류)와 "몸(에이전트)+머리(모델)" 구조**를 얹어 다음 주 설치
> 실습의 지도를 미리 그리게 한다. W02 에서 그 그림을 실제로 조립하고(ssh → Hermes → 오픈 모델),
> W03 은 **도구 없이 브라우저로만** 해킹의 기본기를 손에 익힌다 — 사람이 무엇을 보고 판단하는지
> 알아야 W04 에서 에이전트가 하는 일을 알아볼 수 있기 때문이다. W04 는 에이전트가 운전하되
> **매 단계 학생이 브라우저로 교차 검증**한다(환각 대응). W05 는 스스로 찾는 대결.
> 특별 세션 3주는 표적을 바꿔 **AI 서비스 자체**를 뚫고, 마지막에 방어자로 전환한다.

  `hello.txt` 생성으로 반드시 검증하고, 안 되면 모델을 교체한다.

### W03 — 웹의 작동 원리 + 직접 해보는 웹 해킹 (7h)
- 요청/응답·쿠키·세션·DB·개발자도구 → OWASP Top 10 → DVWA 5대 공격.
- **100% 브라우저.** 웹셸 파일 만들기도 텍스트 편집기 방법을 병기.
- DVWA **Security: Low** 고정. 변수 없이 똑같은 결과.

### W04 — AI 에이전트와 함께하는 모의해킹 NeoBank (6h)
- 프롬프트 10단계(정찰 → IDOR → SQLi → 인증우회 → XSS → 권한상승 → 보고서).
- **모든 단계에 "🖱️ 브라우저로 직접 확인" 절을 둔다.** 주소창 / 화면 폼 / F12 콘솔 세 가지 도구.
- 에이전트 보고가 브라우저로 재현되지 않으면 **환각** — 그것을 잡아내는 것도 학습 목표.

### W05 — mini-CTF (MediForum + CTFd) (4h+)
- 표적: **MediForum**(앞에서 한 번도 안 쓴 사이트). 6문제 850점.
- **브라우저 출발점이 문제마다 명시**된다: robots.txt / F12 Network / 주소창 번호 / Cookies /
  신고 버튼. "주소를 추측하세요" 같은 요구는 없다.
- 강사 전용 풀이: `solutions/` (강좌 사이트에서 admin 로그인 시 `/solutions`).
- 수업 전 `ctf/verify_ctf.py --submit` 으로 표적↔정답표↔CTFd 3자 일치를 반드시 확인.

### 특별 1~3 — AI 서비스 모의해킹 (각 5h)
- 표적: **AICompanion**(`:3005`), 기본 **mock 모드** — 진짜 모델·GPU·인터넷 없이 전부 실습.
- 특별 1: 구조 이해 + 정찰(지식베이스 PII·디버그 프롬프트 유출·데이터 덤프) + 우선순위표.
- 특별 2: 직접 인젝션 · 탈옥 · **간접 인젝션(RAG 오염)** · 저장형 인젝션 · 출력 처리(LLM02).
- 특별 3: **과도한 에이전시**(코드 실행·SSRF) · 권한 상승 · 모델 절취 · **방어 설계 + 보고서**.
- mock 은 한국어·영어 공격 문장을 모두 인식한다(원본 el34 버전에서 한국어 트리거를 추가).
