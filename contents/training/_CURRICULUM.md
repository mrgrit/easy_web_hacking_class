# 커리큘럼 설계서 — 5섹터 ↔ 9모듈 매핑

> 원래 요청한 9개 모듈을, "너무 길어지지 않게" **5개 섹터(주차)** 로 묶었습니다.
> 각 섹터 = 교과서(`lecture_weekNN.md`) + 실습(`lab_weekNN.yaml`) 한 쌍.

## 한눈에 보기

| 섹터 | 제목 | 합친 원본 모듈 | 시간 | 난이도 곡선 |
|------|------|----------------|------|-------------|
| **W01** | AI와 AI 에이전트, 그리고 윤리 | ① AI개론+에이전트 ② AI·보안 윤리 | 4h | 🟢 아주 쉬움(흥미 유발) |
| **W02** | 리눅스·Claude Code 첫걸음 + 내 첫 웹사이트 + GitHub | ③ 리눅스/CC ⑤ CC 체험(심리테스트) ⑥ GitHub | 4h | 🟢 친숙하게 |
| **W03** | 웹의 작동 원리 + 직접 해보는 웹 해킹(DVWA) | ④ 웹 기초 ⑦ 웹 해킹 기법·사례 | 7h | 🟡 천천히 올라감 |
| **W04** | AI 에이전트와 함께하는 모의해킹(NeoBank) | ⑧ AI 활용 모의해킹 | 6h | 🟠 몰입·자동화 |
| **W05** | 🏁 mini-CTF(MediForum + CTFd) | ⑨ mini-CTF | 4h+ | 🔴 응용·대결 |

> **설계 의도** — 가장 노잼이 되기 쉬운 *개론·윤리(①②)* 를 맨 앞에 짧고 강렬하게 배치하고,
> 곧바로 *직접 만들기(⑤)* 의 성취감으로 끌고 갑니다. 웹 기초(④)는 포기가 속출하는 구간이라
> 바로 다음의 *해킹 체험(⑦)* 과 한 섹터로 묶어 "이걸 배우면 바로 이런 걸 뚫는다"는 보상을
> 즉시 연결했습니다. 마지막 ⑧⑨는 AI 에이전트가 운전대를 잡는 클라이맥스입니다.

| 시간 | 내용 | 유형 |
|------|------|------|
| 0:00–0:50 | AI가 뭐길래? — 체감 데모(이미지/글 생성), "예측 기계" 비유 | 이론(가벼움) |
| 0:50–1:40 | AI 에이전트 = 스스로 도구를 쓰는 AI. Claude Code 시연 1분 | 이론+시연 |
| 1:40–2:00 | 휴식 | — |
| 2:00–2:50 | AI 윤리 — 환각/편향/저작권/딥페이크, 우리 반 토론 | 토론 |
| 2:50–3:40 | 사이버보안 윤리 — 해킹은 언제 범죄인가(정보통신망법), 화이트해커 | 토론 |
| 3:40–4:00 | "보안 서약서" 작성 + 다음 주 예고 | 정리 |
